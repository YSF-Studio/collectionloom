# Cursor Prompt: Export Module — CollectionLoom V1

## Tujuan
Implementasi Rust export module untuk menghasilkan paket handover dari snapshot dan diff CollectionLoom. Tiga format: JSON pack, Markdown report, ZIP bundle.

## Aturan Umum
- Semua ekspor deterministic (input sama → output byte sama).
- Output ditulis ke folder `exports/<case_id>/`.
- Setiap ekspor menyertakan hash manifest.
- Wajib handle error dengan jelas (disk full, path invalid, dll).

## Fungsi Inti
```rust
pub struct ExportEngine;

impl ExportEngine {
    /// Ekspor normalized JSON pack
    pub fn export_json_pack(case: &Case, snapshot: &SnapshotMeta, artifacts_dir: &Path, output_path: &Path) -> ExportResult;

    /// Ekspor markdown summary report
    pub fn export_markdown_report(case: &Case, snapshot: &SnapshotMeta, diff: Option<&DiffResult>, output_path: &Path) -> ExportResult;

    /// Buat ZIP bundle lengkap
    pub fn export_zip_bundle(case_dir: &Path, output_path: &Path) -> ExportResult;
}
```

## 1. JSON Pack (`evidence_pack.json`)
Schema:
```json
{
  "schema_version": "1.0.0",
  "export_type": "json_pack",
  "exported_at": "ISO8601",
  "case": { /* case.json content */ },
  "snapshot": { /* snapshot_meta.json content */ },
  "artifacts": {
    "system": { /* parsed system.json */ },
    "process": [ /* parsed process.json — hanya top 100 untuk V1 */ ],
    "network": { /* parsed */ },
    "autoruns": [ /* parsed */ ],
    "users": [ /* parsed */ ]
  },
  "integrity": {
    "manifest_hash": "sha256 dari hash_manifest.json",
    "pack_hash": "sha256 dari file ini (self-hash)"
  }
}
```

Aturan:
- Jangan embed raw binary — hanya data JSON yang sudah dikumpulkan.
- Process list: hanya top 100 berdasarkan CPU% (V1).
- Artifact yang corrupt atau tidak terbaca: field `"_error": "reason"`.

## 2. Markdown Report (`case_report.md`)
Template:
```markdown
# CollectionLoom Case Report

## Case Info
- **Case ID:** `{case_id}`
- **Title:** {title}
- **Operator:** {operator_name}
- **Created:** {created_at}
- **Status:** {status}

## Snapshot Summary
- **Snapshot ID:** `{snapshot_id}`
- **Host:** {hostname}
- **OS:** {os_family} {os_version}
- **Profile:** {profile}
- **Started:** {started_at}
- **Duration:** {duration_seconds}s
- **Status:** {status}

## Collector Modules
| Module | Status | Items | Duration |
|--------|--------|-------|----------|
{module_rows}

## Artifact Integrity
- **Total files:** {total_files}
- **Manifest hash:** `{manifest_hash}`
- **Verification:** {passed/failed}

## Changes (if diff available)
### Added ({total_added})
{added_items}

### Removed ({total_removed})
{removed_items}

### Changed ({total_changed})
{changed_items}

---
*Generated by CollectionLoom v{version} at {exported_at}*
```

## 3. ZIP Bundle (`collection_bundle.zip`)
Isi ZIP:
```
collection_bundle/
├── case.json
├── snapshot_meta.json
├── artifacts/
│   ├── system.json
│   ├── process.json
│   ├── network.json
│   ├── autoruns.json
│   └── users.json
├── hash_manifest.json
├── collector_audit.log
├── evidence_pack.json        (jika ada)
└── case_report.md            (jika ada)
```

Gunakan `zip` crate atau `tar` jika zip bermasalah.

## ExportResult
```rust
pub struct ExportResult {
    pub export_type: ExportType, // JsonPack | MarkdownReport | ZipBundle
    pub output_path: PathBuf,
    pub size_bytes: u64,
    pub sha256: String,
    pub exported_at: String,
    pub error: Option<String>,
}

pub enum ExportType {
    JsonPack,
    MarkdownReport,
    ZipBundle,
}
```

## Output yang Diharapkan
- Semua fungsi export compile dan menghasilkan output sesuai format.
- Markdown report human-readable dan siap lapangan.
- ZIP bundle diekstrak dengan struktur folder yang benar.
- Unit test: generate export → verify isi file → verify hash cocok.

## Catatan
- Jangan compress JSON/teks di ZIP (store mode lebih cepat, V1 priority speed over size).
- Markdown report bisa ditulis manual string builder (jangan pakai template engine).
- Test dengan folder case yang sudah ada snapshot + diff.
